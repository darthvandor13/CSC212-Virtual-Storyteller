◆ apply_known_metadata()

CoreSchema None pydantic._internal._known_annotated_metadata.apply_known_metadata ( Any  annotation,
CoreSchema  schema 
) 
Apply `annotation` to `schema` if it is an annotation we know about (Gt, Le, etc.).
Otherwise return `None`.

This does not handle all known annotations. If / when it does, it can always
return a CoreSchema and return the unmodified schema if the annotation should be ignored.

Assumes that GroupedMetadata has already been expanded via `expand_grouped_metadata`.

Args:
    annotation: The annotation.
    schema: The schema.

Returns:
    An updated schema with annotation if it is an annotation we know about, `None` otherwise.

Raises:
    PydanticCustomError: If `Predicate` fails.

Iterable[str]  allowed,
Any  source_type 
) 
A small utility function to validate that the given metadata can be applied to the target.
More than saving lines of code, this gives us a consistent error message for all of our internal implementations.

Args:
    metadata: A dict of metadata.
    allowed: An iterable of allowed metadata.
    source_type: The source type.

Raises:
    TypeError: If there is metadatas that can't be applied on source type.

◆ collect_known_metadata()

tuple[dict[str, Any], list[Any]] pydantic._internal._known_annotated_metadata.collect_known_metadata ( Iterable[Any]  annotations) 
Split `annotations` into known metadata and unknown annotations.

Args:
    annotations: An iterable of annotations.

Returns:
    A tuple contains a dict of known metadata and a list of unknown annotations.

Example:
    ```python
    from annotated_types import Gt, Len

    from pydantic._internal._known_annotated_metadata import collect_known_metadata

    print(collect_known_metadata([Gt(1), Len(42), ...]))
    #> ({'gt': 1, 'min_length': 42}, [Ellipsis])
    ```

Example:

